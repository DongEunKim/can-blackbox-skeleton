# CAN Blackbox 개발 계획

## 1. 프로젝트 환경 구성 검토

### 1.1 기술 스택

| 구분 | 기술 | 비고 |
|------|------|------|
| 언어 | Python 3.9+ | |
| CAN 통신 | python-can | BLF 지원, CAN 로깅에 필수 |
| 설정 관리 | configparser | 표준 라이브러리 |
| AWS 연동 | AWS IoT Device SDK v2 (Python) | Greengrass StreamManager 연동용 |
| 파일 감시 | pathlib + threading + time | 표준 라이브러리로 폴링 방식 구현 |
| 파일 처리 | os, pathlib, shutil | 표준 라이브러리 |

### 1.2 의존성 (최소화)

```
python-can>=4.3.0
awsiotsdk>=1.12.0
```

- **boto3 미사용**: S3 업로드는 StreamManager가 직접 처리
- **watchdog 미사용**: pathlib 기반 폴링으로 신규 파일 감지

### 1.3 프로젝트 구조

```
can-blackbox/
├── config.ini
├── requirements.txt
├── src/
│   ├── can_logger.py         # CAN → BLF 로깅
│   ├── file_watcher.py       # 폴더 감시 (폴링, stdlib)
│   ├── uploader.py           # StreamManager → S3 전달
│   ├── storage_manager.py    # 용량 관리, 오래된 파일 삭제
│   └── stream_manager_mock.py  # StreamManager 가상화
├── tests/
│   └── ...
├── docs/
│   └── 개발계획.md
└── scripts/
    └── virtual_can_setup.sh
```

## 2. 단계별 구현 계획

| 단계 | 작업 | 산출물 | 검증 |
|------|------|--------|------|
| **1** | INI 설정 구조 정의 및 로더 | `config.ini`, 설정 로더 | 설정 파싱 단위 테스트 |
| **2** | Virtual CAN 구성 및 더미 데이터 | vcan0 설정, candump/cangen | vcan0 동작 확인 |
| **3** | CAN → BLF 로깅 (5분 로테이션) | `can_logger.py` | BLF 생성·로테이션 테스트 |
| **4** | 폴더 감시 (표준 라이브러리 폴링) | `file_watcher.py` | 신규 파일 감지 테스트 |
| **5** | StreamManager 모킹 | `stream_manager_mock.py` | API 호환 검증 |
| **6** | StreamManager로 파일 전달 (S3 업로드) | `uploader.py` | 업로드 시뮬레이션 테스트 |
| **7** | 용량 초과 시 오래된 파일 삭제 | `storage_manager.py` | 용량 관리·삭제 테스트 |
| **8** | 전체 통합 및 E2E 테스트 | 통합 파이프라인 | vcan → BLF → 감시 → 전달 → 삭제 검증 |

## 3. 시스템 요구사양서 (개발 기준)

### 3.1 기능 요구사항

- **FR-01**: CAN 버스 메시지를 BLF 형식으로 연속 로깅
- **FR-02**: 로그 파일을 5분 단위로 로테이션하여 지정 폴더에 저장
- **FR-03**: 저장 폴더에 새 파일 생성 시 감지 (표준 라이브러리만 사용)
- **FR-04**: 감지된 파일을 StreamManager로 전달하여 S3에 업로드
- **FR-05**: S3 전달 완료 후 해당 로컬 파일 삭제
- **FR-06**: 총 용량 초과 시 오래된 파일 순서로 삭제

### 3.2 비기능 요구사항

- **NFR-01**: 설정은 INI 파일로 관리
- **NFR-02**: 테스트 시 Virtual CAN 및 더미 데이터 사용
- **NFR-03**: StreamManager 미설치 환경에서 모킹 사용
- **NFR-04**: 외부 패키지는 python-can, awsiotsdk만 사용

### 3.3 인터페이스

- **CAN**: `can0` / `vcan0` (INI 설정)
- **파일**: BLF 형식
- **저장 경로**: INI로 지정
- **S3**: StreamManager API로 파일 전달 (boto3 미사용)

## 4. 구현 상세 (표준 라이브러리 활용)

### 4.1 파일 감시 (watchdog 대체)

- `pathlib.Path.iterdir()`로 주기적 폴더 스캔
- `time.sleep()` 또는 `threading.Timer`로 폴링 주기 제어
- 사용 모듈: `pathlib`, `time`, `threading`, `os`

### 4.2 S3 업로드

- StreamManager `AppendMessage` 등으로 파일을 S3 스트림에 전달
- boto3 없이 awsiotsdk의 StreamManager API만 사용
