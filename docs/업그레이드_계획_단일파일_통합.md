## 2. 참조 구조 (aws-greengrass-labs-s3-file-uploader)

```python
# DirectoryUploader.py - 약 220줄, 단일 파일
class DirectoryUploader:
    # Stream Manager SDK 직접 import
    # __init__: 스트림 생성, 스트림 삭제 후 재생성
    # __scan: glob + mtime 정렬, active file 제외, append S3 task
    # __processStatus: status stream 읽기, Success 시 삭제, Failure 시 filesProcessed 제외
    # Run: asyncio gather(__scan, __processStatus)
    # Close: client.close()
```

**특징**: stream_manager SDK를 직접 사용, 별도 래퍼 없음, 모킹 없음(Greengrass 전제)

## 3. CAN Blackbox 단일 파일 설계

### 3.1 차이점 반영

| 항목 | s3-file-uploader | CAN Blackbox |
|------|------------------|--------------|
| 파일 감지 | glob + active 제외 | 크기 안정화 (min_stable_polls) |
| 모킹 | 없음 | use_mock 시 로컬 mock_uploads |
| S3 키 | prefix/filename | prefix + !{timestamp:yyyy/MM/dd}/filename |
| 저장소 정리 | 없음 | max_total_mb 초과 시 삭제 |
| 스트림 | 시작 시 삭제 | 기존 유지 (재시작 시 큐 보존) |

### 3.2 단일 파일 구조: directory_uploader.py (현재 구현)

```
directory_uploader.py
├── 상수: BLF_SUFFIX
├── 스캔: _scan_new_files (0바이트 제외, 크기 안정화)
├── 저장소: get_total_size_mb, trim_storage
├── 모킹: MockUploadClient (use_mock 시 로컬 bucket/prefix/날짜 구조)
├── 실제: _create_real_upload_client, create_upload_client
├── DirectoryUploader 클래스
│   ├── __init__(watch_dir, client, max_total_mb, poll_interval, min_stable_polls)
│   ├── _scan(), _trim_storage(), run(), close()
└── main() / if __name__ == "__main__": config 로드, 재시작 루프, DirectoryUploader 실행
```

### 3.3 포함할 로직 (인라인)

| 기능 | 현재 위치 | 통합 후 |
|------|----------|---------|
| 스캔 (0바이트 제외, 크기 안정) | file_watcher._scan_new_files | directory_uploader 내부 함수 |
| 용량 계산 | storage_manager.get_total_size_mb | directory_uploader 내부 함수 |
| 용량 정리 | storage_manager.trim_storage | directory_uploader 내부 함수 |
| Stream Manager | stream_manager_real | directory_uploader 내부 (use_mock=false) |
| 모킹 | stream_manager_mock | directory_uploader 내부 (use_mock=true) |

### 3.4 config 의존

- `config_loader`는 유지 (다른 모듈에서도 사용)
- directory_uploader는 `get_logging_config`, `get_storage_config`, `get_stream_manager_config`, `get_watcher_config` 사용
